<h2 class="text-2xl font-poppins font-bold mb-6 text-anchor-navy">1. Understanding Core Web Vitals</h2>

<p class="text-lg mb-4">Google's Core Web Vitals are the foundation of modern page speed metrics:</p>

<div class="bg-sand-beige p-6 rounded-lg mb-6">
<h3 class="text-xl font-poppins font-bold mb-4 text-anchor-navy">⚡ Core Web Vitals Explained</h3>
<ul class="space-y-2 text-lg">
<li><strong>Largest Contentful Paint (LCP):</strong> Measures loading performance (should be under 2.5 seconds)</li>
<li><strong>First Input Delay (FID):</strong> Measures interactivity (should be under 100 milliseconds)</li>
<li><strong>Cumulative Layout Shift (CLS):</strong> Measures visual stability (should be under 0.1)</li>
</ul>
</div>

<h2 class="text-2xl font-poppins font-bold mb-6 text-anchor-navy">2. Image Optimization Strategies</h2>

<h3 class="text-xl font-poppins font-bold mb-4 text-anchor-navy">Choose the Right Format</h3>

<p class="text-lg mb-4">Image format selection can significantly impact loading speed:</p>

<ul class="list-disc list-inside mb-6 space-y-2 text-lg">
<li><strong>WebP format:</strong> 25-35% smaller than JPEG while maintaining quality</li>
<li><strong>AVIF format:</strong> Even smaller than WebP, but browser support is growing</li>
<li><strong>Progressive JPEGs:</strong> Load gradually, improving perceived performance</li>
<li><strong>SVG for graphics:</strong> Scalable and lightweight for icons and logos</li>
<li><strong>Responsive images:</strong> Serve different sizes for different devices</li>
</ul>

<h3 class="text-xl font-poppins font-bold mb-4 text-anchor-navy">Compression and Sizing</h3>

<ul class="list-disc list-inside mb-6 space-y-2 text-lg">
<li><strong>Lossless compression:</strong> Reduce file size without quality loss</li>
<li><strong>Appropriate dimensions:</strong> Don't use larger images than needed</li>
<li><strong>Lazy loading:</strong> Load images only when they're about to be viewed</li>
<li><strong>CDN delivery:</strong> Serve images from edge locations</li>
<li><strong>Image sprites:</strong> Combine multiple small images into one file</li>
</ul>

<h2 class="text-2xl font-poppins font-bold mb-6 text-anchor-navy">3. Code Optimization Techniques</h2>

<h3 class="text-xl font-poppins font-bold mb-4 text-anchor-navy">CSS and JavaScript Optimization</h3>

<ul class="list-disc list-inside mb-6 space-y-2 text-lg">
<li><strong>Minification:</strong> Remove unnecessary characters from code</li>
<li><strong>Concatenation:</strong> Combine multiple files into single files</li>
<li><strong>Critical CSS:</strong> Inline above-the-fold styles</li>
<li><strong>Async loading:</strong> Load non-critical JavaScript asynchronously</li>
<li><strong>Defer non-essential scripts:</strong> Delay loading until page is ready</li>
</ul>

<h3 class="text-xl font-poppins font-bold mb-4 text-anchor-navy">HTML Optimization</h3>

<ul class="list-disc list-inside mb-6 space-y-2 text-lg">
<li><strong>Clean markup:</strong> Remove unnecessary HTML elements</li>
<li><strong>Semantic structure:</strong> Use proper HTML5 semantic elements</li>
<li><strong>Efficient selectors:</strong> Use CSS selectors that render quickly</li>
<li><strong>Reduce DOM size:</strong> Minimize the number of elements on the page</li>
<li><strong>Optimize meta tags:</strong> Keep meta descriptions concise</li>
</ul>

<h2 class="text-2xl font-poppins font-bold mb-6 text-anchor-navy">4. Server and Hosting Optimization</h2>

<h3 class="text-xl font-poppins font-bold mb-4 text-anchor-navy">Hosting Considerations</h3>

<p class="text-lg mb-4">Your hosting choice significantly impacts website speed:</p>

<ul class="list-disc list-inside mb-6 space-y-2 text-lg">
<li><strong>Australian hosting:</strong> Choose servers located in Australia for faster local loading</li>
<li><strong>SSD storage:</strong> Solid-state drives are much faster than traditional HDDs</li>
<li><strong>CDN implementation:</strong> Use content delivery networks for global speed</li>
<li><strong>Server response time:</strong> Aim for under 200ms server response time</li>
<li><strong>Uptime guarantees:</strong> Choose hosting with 99.9%+ uptime</li>
</ul>

<h3 class="text-xl font-poppins font-bold mb-4 text-anchor-navy">Caching Strategies</h3>

<ul class="list-disc list-inside mb-6 space-y-2 text-lg">
<li><strong>Browser caching:</strong> Set appropriate cache headers for static resources</li>
<li><strong>Server-side caching:</strong> Implement page and object caching</li>
<li><strong>Database caching:</strong> Cache database queries and results</li>
<li><strong>CDN caching:</strong> Leverage CDN edge caching for faster delivery</li>
<li><strong>Application caching:</strong> Cache frequently accessed data in memory</li>
</ul>

<h2 class="text-2xl font-poppins font-bold mb-6 text-anchor-navy">5. Database and Backend Optimization</h2>

<p class="text-lg mb-4">Backend performance directly affects frontend speed:</p>

<ul class="list-disc list-inside mb-6 space-y-2 text-lg">
<li><strong>Database optimization:</strong> Optimize queries and indexes</li>
<li><strong>Connection pooling:</strong> Reuse database connections efficiently</li>
<li><strong>Query optimization:</strong> Minimize database round trips</li>
<li><strong>Data compression:</strong> Compress data in transit</li>
<li><strong>Background processing:</strong> Move heavy tasks to background jobs</li>
</ul>

<h2 class="text-2xl font-poppins font-bold mb-6 text-anchor-navy">6. Mobile Speed Optimization</h2>

<p class="text-lg mb-4">Mobile users have different speed requirements:</p>

<ul class="list-disc list-inside mb-6 space-y-2 text-lg">
<li><strong>Mobile-first design:</strong> Design for mobile performance first</li>
<li><strong>Touch-friendly optimization:</strong> Optimize for touch interactions</li>
<li><strong>Mobile-specific images:</strong> Serve appropriately sized images for mobile</li>
<li><strong>Progressive web apps:</strong> Consider PWA features for better mobile experience</li>
<li><strong>Mobile network optimization:</strong> Optimize for slower mobile connections</li>
</ul>

<h2 class="text-2xl font-poppins font-bold mb-6 text-anchor-navy">7. Third-Party Script Management</h2>

<p class="text-lg mb-4">Third-party scripts can significantly slow down your site:</p>

<ul class="list-disc list-inside mb-6 space-y-2 text-lg">
<li><strong>Audit third-party scripts:</strong> Identify and evaluate all external scripts</li>
<li><strong>Load asynchronously:</strong> Load non-critical scripts asynchronously</li>
<li><strong>Use resource hints:</strong> Implement preconnect and dns-prefetch</li>
<li><strong>Monitor performance impact:</strong> Track how third-party scripts affect speed</li>
<li><strong>Consider alternatives:</strong> Look for lighter alternatives to heavy scripts</li>
</ul>

<li><strong>First Input Delay (FID):</strong> Interactivity responsiveness</li>
<li><strong>Cumulative Layout Shift (CLS):</strong> Visual stability</li>
</ul>

<h2 class="text-2xl font-poppins font-bold mb-6 text-anchor-navy">9. Advanced Optimization Techniques</h2>

<h3 class="text-xl font-poppins font-bold mb-4 text-anchor-navy">HTTP/2 and HTTP/3</h3>

<ul class="list-disc list-inside mb-6 space-y-2 text-lg">
<li><strong>HTTP/2 implementation:</strong> Enable multiplexing and server push</li>
<li><strong>HTTP/3 adoption:</strong> Consider QUIC protocol for faster connections</li>
<li><strong>Server push:</strong> Proactively send critical resources</li>
<li><strong>Header compression:</strong> Reduce HTTP header overhead</li>
<li><strong>Connection multiplexing:</strong> Handle multiple requests simultaneously</li>
</ul>

<h3 class="text-xl font-poppins font-bold mb-4 text-anchor-navy">Resource Hints</h3>

<ul class="list-disc list-inside mb-6 space-y-2 text-lg">
<li><strong>DNS prefetch:</strong> Resolve domain names early</li>
<li><strong>Preconnect:</strong> Establish connections to external domains</li>
<li><strong>Preload:</strong> Load critical resources early</li>
<li><strong>Prefetch:</strong> Load resources for likely future navigation</li>
<li><strong>Prerender:</strong> Render entire pages in background</li>
</ul>

<ul class="list-disc list-inside mb-6 space-y-2 text-lg">

<li><strong>Ignoring mobile:</strong> Only optimizing for desktop performance</li>
<li><strong>Poor image optimization:</strong> Using wrong formats or sizes</li>
<li><strong>Excessive third-party scripts:</strong> Loading too many external resources</li>
<li><strong>Inadequate caching:</strong> Not implementing proper caching strategies</li>
<li><strong>Server location:</strong> Hosting far from target audience</li>
</ul>
